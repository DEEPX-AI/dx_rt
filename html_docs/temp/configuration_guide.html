    Programming Guide: dxrt::Configuration¶

    1. Overview¶

    The dxrt::Configuration class is a central component within the DXRT library responsible for managing global application settings. It is designed as a thread-safe singleton, ensuring that there is only one instance of the configuration manager throughout the application's lifecycle. This provides a consistent and safe point of access for querying and modifying configuration parameters from any part of the codebase, even in multi-threaded environments.

    Key Features:

    • Singleton Pattern: Guarantees a single, globally accessible instance.
    • Thread Safety: Integrated mutex protects against race conditions during concurrent access.
    • Dynamic Configuration: Allows enabling/disabling features and setting specific attributes at runtime.
    • Configuration Locking: Supports making settings read-only to prevent changes after initialization.
    • Version Reporting: Provides methods to retrieve versions of the library, drivers, and firmware.

    2. Core Concepts¶

    Singleton Pattern¶

    The Configuration class cannot be instantiated directly. Instead, you must access its single instance through the static GetInstance() method. This design ensures that all parts of the application refer to the same set of configuration data.

    // Correct way to get the configuration instance
dxrt::Configuration& config = dxrt::Configuration::GetInstance();

// Incorrect - The following lines will cause a compile error
// dxrt::Configuration myConfig; // Error: constructor is private
// dxrt::Configuration* pConfig = new dxrt::Configuration(); // Error: constructor is private

    Thread Safety¶

    All public methods that modify or read configuration data (e.g., SetEnable, GetEnable, SetAttribute) are internally protected by a std::mutex. This makes it safe to call them from multiple threads without needing external locking mechanisms, preventing data corruption and race conditions.

    Items and Attributes¶

    Configuration is organized around two main concepts:

    • ITEM: An enumeration representing a configurable feature or module (e.g., DEBUG, PROFILER). These items can typically be enabled or disabled.
    • ATTRIBUTE: An enumeration representing a specific property of an ITEM. Attributes hold string-based values that provide more detailed settings beyond a simple on/off state (e.g., a file path for PROFILER_SAVE_DATA).

    3. Getting Started: Obtaining the Instance¶

    To begin working with the configuration, first retrieve the singleton instance. It's recommended to get a reference to it once and reuse it.

    #include "dxrt/common.h" // Assuming Configuration is in this header
#include <iostream>

int main() {
    // Get the single, global instance of the Configuration class
    dxrt::Configuration& config = dxrt::Configuration::GetInstance();

    // You can now use the 'config' object to manage settings
    std::cout << "Successfully obtained Configuration instance." << std::endl;

    return 0;
}

    4. Key Operations and Code Examples¶

    4.1. Loading Configuration from a File¶

    You can initialize settings from an external file using LoadConfigFile(). The specific format of the file is implementation-dependent, but it typically allows you to define which items are enabled and what their attribute values are.

    dxrt::Configuration& config = dxrt::Configuration::GetInstance();
try {
    // Load settings from a configuration file
    config.LoadConfigFile("settings.ini");
} catch (const std::runtime_error& e) {
    std::cerr << "Failed to load config file: " << e.what() << std::endl;
}

    4.2. Enabling and Disabling Features¶

    You can turn features on or off using SetEnable() and check their status with GetEnable().

    dxrt::Configuration& config = dxrt::Configuration::GetInstance();

// Enable the profiler
config.SetEnable(dxrt::Configuration::ITEM::PROFILER, true);

// Check if the profiler is enabled
if (config.GetEnable(dxrt::Configuration::ITEM::PROFILER)) {
    std::cout << "Profiler is enabled." << std::endl;
} else {
    std::cout << "Profiler is disabled." << std::endl;
}

// Disable debug mode
config.SetEnable(dxrt::Configuration::ITEM::DEBUG, false);

    4.3. Working with Attributes¶

    For more fine-grained control, use SetAttribute() and GetAttribute() to manage string-based values for an item's attributes.

    dxrt::Configuration& config = dxrt::Configuration::GetInstance();

// First, ensure the parent item is enabled
config.SetEnable(dxrt::Configuration::ITEM::PROFILER, true);

// Set the path where profiler data should be saved
std::string profile_path = "/var/log/my_app_profile.json";
config.SetAttribute(dxrt::Configuration::ITEM::PROFILER,
                    dxrt::Configuration::ATTRIBUTE::PROFILER_SAVE_DATA,
                    profile_path);

// Retrieve the attribute value later
std::string saved_path = config.GetAttribute(dxrt::Configuration::ITEM::PROFILER,
                                          dxrt::Configuration::ATTRIBUTE::PROFILER_SAVE_DATA);

if (!saved_path.empty()) {
    std::cout << "Profiler data will be saved to: " << saved_path << std::endl;
}

    4.4. Locking Configuration¶

    After initial setup, you can lock an item to prevent any further changes to its enabled status or attributes. This is useful for ensuring configuration stability during the main application logic.

    dxrt::Configuration& config = dxrt::Configuration::GetInstance();

// Set initial configuration for the profiler
config.SetEnable(dxrt::Configuration::ITEM::PROFILER, true);

// Lock the PROFILER item
config.LockEnable(dxrt::Configuration::ITEM::PROFILER);

// Any subsequent attempt to change this item will be ignored (silently)
config.SetEnable(dxrt::Configuration::ITEM::PROFILER, false); // This will have no effect

// The profiler remains enabled
bool is_enabled = config.GetEnable(dxrt::Configuration::ITEM::PROFILER); // will return true

    4.5. Retrieving Version Information¶

    The class provides several methods to query version information, which is critical for debugging, logging, and ensuring system compatibility. These methods may throw an exception if a version mismatch or other error occurs.

    #include <vector>
#include <utility>
#include <string>

dxrt::Configuration& config = dxrt::Configuration::GetInstance();

try {
    std::cout << "DXRT Library Version: " << config.GetVersion() << std::endl;
    std::cout << "Driver Version: " << config.GetDriverVersion() << std::endl;
    std::cout << "ONNX Runtime Version: " << config.GetONNXRuntimeVersion() << std::endl;

    std::vector<std::pair<int, std::string>> fw_versions = config.GetFirmwareVersions();
    for (const auto& fw : fw_versions) {
        std::cout << "Device " << fw.first << " Firmware Version: " << fw.second << std::endl;
    }

} catch (const std::runtime_error& e) { // Or a more specific exception type if defined
    std::cerr << "Error retrieving version information: " << e.what() << std::endl;
}

    5. API Reference Summary¶

    Main Methods¶

    Method Signature Description
    static Configuration& GetInstance() Returns the unique singleton instance of the class.
    void LoadConfigFile(const std::string& fileName) Loads and applies configurations from a specified file.
    void SetEnable(const ITEM item, bool enabled) Sets the enabled (true) or disabled (false) status for an item.
    bool GetEnable(const ITEM item) Retrieves the current enabled status of an item.
    void SetAttribute(const ITEM item, const ATTRIBUTE attrib, const std::string& value) Sets a string value for a specific attribute of an item.
    std::string GetAttribute(const ITEM item, const ATTRIBUTE attrib) Retrieves the string value of an attribute. Returns empty if not found.
    void LockEnable(const ITEM item) Locks an item's configuration, making it read-only.
    std::string GetVersion() const Gets the version of the DXRT library.
    std::string GetDriverVersion() const Gets the version of the device driver.
    std::string GetPCIeDriverVersion() const Gets the version of the PCIe driver.
    std::vector<std::pair<int, std::string>> GetFirmwareVersions() const Gets the firmware versions for all detected devices.
    std::string GetONNXRuntimeVersion() const Gets the version of the linked ONNX Runtime library.

    Enumerations¶

    enum class ITEM¶

    Enumerator Description
    DEBUG Configuration related to debug mode.
    PROFILER Configuration related to profiler functionality.
    SERVICE Configuration related to service operation.
    DYNAMIC_CPU_THREAD Configuration for dynamic CPU thread management.
    TASK_FLOW Configuration related to task flow management.
    SHOW_THROTTLING Whether to display throttling information.
    SHOW_PROFILE Whether to display profile information.
    SHOW_MODEL_INFO Whether to display model information.

    enum class ATTRIBUTE¶

    Enumerator Description
    PROFILER_SHOW_DATA Attribute for showing profiler data.
    PROFILER_SAVE_DATA Attribute for saving profiler data.
