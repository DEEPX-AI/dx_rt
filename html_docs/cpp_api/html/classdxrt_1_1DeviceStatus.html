Detailed Description

A class that provides an abstraction of device information.

This class encapsulates various details about the device, including specifications, clock speeds, and temperature. It allows users to easily retrieve device status and characteristics.

Example usage:

// Retrieve device status information
auto statInfo = GetCurrentStatus(DevicePtr device);
cout << GetInfoString() << endl << GetStatusString() << endl;

◆ DdrStatusStr()

std::string dxrt::DeviceStatus::DdrStatusStr ( int  ch) const

Retrieves the status of a specified LPDDR memory channel.

This function provides information about an LPDDR channel, including its configuration and real-time status. The information is based on memory register values (MR registers) that are relevant to LPDDR temperature and operation.

Example Output: 

LPDDR Channel 0: MR4=0x1A, Temperature Normal

Explanation of the format:

  • LPDDR Channel 0 -> Indicates the specific LPDDR memory channel (0, 1, 2, or 3).
  • MR4=0x1A -> Shows the MR4 register value, which relates to temperature monitoring.
  • Temperature Normal -> Provides an interpretation of the temperature status.

This function is useful for debugging memory performance and monitoring thermal conditions.

Parameters
chThe LPDDR memory channel index (0 to 3).
Returns
A formatted string containing the status of the specified LPDDR memory channel.

◆ DvfsStateInfoStr()

std::string dxrt::DeviceStatus::DvfsStateInfoStr ( ) const

Retrieves the current Dynamic Voltage and Frequency Scaling (DVFS) state of the device.

This function provides information about whether DVFS is enabled or disabled. When enabled, it dynamically adjusts voltage and clock speed to optimize power consumption and performance.

The returned string follows one of the following formats:

  • **"dvfs Enabled (Max Frequency: 900)"** -> DVFS is active, with a maximum clock frequency of 900 MHz.
  • **"dvfs Disabled"** -> DVFS is turned off, meaning the voltage and clock speed remain fixed.

This function is useful for power management analysis and performance tuning.

Returns
A string indicating the DVFS state and maximum frequency (if enabled).

Retrieves detailed information about the device.

This function provides a formatted string containing key device specifications, including model name, memory type, board type, and firmware version. It is equivalent to running the dxrt-cli -i command.

The returned string follows this format: 

Device 0: M1, Accelerator type
Memory: LPDDR4 4200 MHz, 1.98 GiB
Board: SOM ASIC, Rev 0.2
2.0.3

Explanation of the format:

  • Device 0: M1, Accelerator type -> Displays the device index, model name (e.g., M1), and type (e.g., Accelerator).
  • Memory: LPDDR4 4200 MHz, 1.98 GiB -> Provides memory type, frequency, and size.
  • Board: SOM ASIC, Rev 0.2 -> Indicates the board type (e.g., System-on-Module) and revision number.
  • 2.0.3 -> Represents the firmware or software version of the device.

This function is useful for obtaining static device properties and verifying hardware configurations.

Returns
A formatted string containing device specifications.

◆ GetStatusString()

std::string dxrt::DeviceStatus::GetStatusString ( ) const

Retrieves the real-time status of the device.

This function returns a formatted string with dynamic device status information, including voltage, clock speed, and temperature for each NPU (Neural Processing Unit), as well as the DVFS (Dynamic Voltage and Frequency Scaling) state. It is equivalent to running the dxrt-cli -s command.

The returned string follows this format: 

NPU 0: voltage 825 mV, clock 1000 MHz, temperature 50'C
NPU 1: voltage 800 mV, clock 600 MHz, temperature 52'C
dvfs Disabled

Explanation of the format:

  • NPU 0: voltage 825 mV, clock 1000 MHz, temperature 50'C -> Displays the voltage, clock frequency, and temperature of NPU 0.
  • NPU 1: voltage 800 mV, clock 600 MHz, temperature 52'C -> Displays the same status information for NPU 1.
  • dvfs Disabled -> Indicates whether DVFS (Dynamic Voltage and Frequency Scaling) is enabled or disabled.

This function is useful for monitoring the device real-time operational status, diagnosing thermal or performance issues, and optimizing power management.

Returns
A formatted string containing real-time device status information.
